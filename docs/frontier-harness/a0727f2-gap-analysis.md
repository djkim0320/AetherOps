## 기존 컴포넌트 재사용 판정

- `durableJobRuntime`: 재사용한다. project FIFO, bounded concurrency, startup recovery와 lease lifecycle의 기반이다.
- `durableJobCheckpointRuntime`: 기준 커밋에는 독립 파일이 삭제되어 없다. 기능은 `durableJobTraceRuntime`, `jobAtomicOperations`, `checkpointRepository`로 분해되었으므로 새 동명 모듈을 만들지 않는다.
- `durableJobTraceRuntime`: fenced trace/event/checkpoint와 SSE publish/readback을 M0~M3 receipt 기반으로 확장한다.
- `durableResumeValidator`: checkpoint/attempt hash 검증은 재사용하되 interrupted false-completion을 먼저 고친다.
- `durableToolExecutionAdapter`: origin link, quarantine, promotion을 M3 VerifiedToolExecutor의 durability adapter로 재사용한다.
- `registerDurableJobHandlers`: public queue semantics는 유지하되 direct Codex bypass를 M3 경로로 수렴한다.
- `ResearchTool`과 static descriptor: M2 compatibility adapter의 입력으로 유지하고 즉시 제거하지 않는다.
- memory/retrieval: ProjectContextBuilder, hybrid scoring, v2 FTS/embedding repository를 재사용하되 legacy production store와 dual source-of-truth를 만들지 않는다.
- Codex OAuth/CLI: provider adapter로 유지하며 core의 provider-specific 타입은 M6에서 축소한다.
- evidence/artifact: stable IDs, origin attempt links, atomic promotion을 유지한다.
- RPC/SSE: API v2와 TanStack Query ownership을 additive 확장한다.
- autonomy verify: M0 AetherBench의 process/artifact/sanitizer/profile 기반으로 재사용한다.

## 단계 게이트

M0~M3가 targeted tests, temp-root migration verify, legacy regression, forced reset, duplicate-side-effect, 전체 offline gate를 모두 통과하기 전에는 M4 이후 production subsystem이나 placeholder schema를 추가하지 않는다. 특히 production research가 아직 `legacy-research.sqlite`를 source-of-truth로 사용하므로 M4는 parity와 단일 cutover 계획 없이 v2 memory dual-write를 시작하지 않는다.
